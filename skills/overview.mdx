## 你有没有这样的困扰

你每天用 Claude 做某件事时，可能会发现自己在**反复说同样的话**——

> "帮我读这份周报，按部门总结，列出风险项……"
> （每周一重复一遍）

> "帮我检查这份方案，按我们的审查清单逐项核对……"
> （每次都要重新说一遍审查标准）

**Agent Skill 就是解决这个问题的。**

## 一句话理解 Skill

用一个类比来理解：

| 概念 | 你的工作里对应的东西 |
|------|---------------------|
| **SOP 文档** | 你们部门的操作手册、检查清单 |
| **Agent Skill** | 把 SOP 变成 Claude 能自动执行的"技能包" |
| **触发 Skill** | 你只需要说一句话（比如"帮我做周报"），Claude 自动按照你的 SOP 流程执行 |

<Note>
  **一句话定义**：Skill 是一份写给 Claude 的"操作手册"，让它按照你定义的步骤、规则和格式来完成任务，而不是每次都从头教它。
</Note>

## Skill 解决什么问题

<Tabs>
  <Tab title="没有 Skill 时">
    - 每次对话你都要重复说明"按什么格式、走什么流程、注意什么规则"
    - 不同的人让 Claude 做同一件事，输出格式和质量参差不齐
    - Claude 可能遗漏你的某些关键要求
    - 新同事需要从头学习"怎么和 Claude 提需求"
  </Tab>
  <Tab title="有了 Skill 之后">
    - 规则和流程写一次，之后自动生效——你只需要说一句触发词
    - 团队所有人的输出格式统一，质量一致
    - 关键规则被固化，不会遗漏
    - 新同事无需学习复杂的提问技巧，直接使用现成的 Skill
  </Tab>
</Tabs>

## Skill 的工作原理（简要说明）

根据 [Anthropic 官方文档](https://platform.claude.com/docs/zh-CN/agents-and-tools/agent-skills/overview)，Skill 采用**渐进式加载**的设计：

| 加载时机 | 内容 | 说明 |
|---------|------|------|
| **始终加载** | Skill 名称和描述 | Claude 启动时就知道有哪些 Skill 可用，但不占用太多空间 |
| **触发时加载** | 操作指南（SKILL.md） | 只有你的请求匹配时，Claude 才读取详细指令 |
| **按需加载** | 模板、参考资料、脚本 | Claude 只在需要时才读取额外的文件 |

这种设计意味着你可以安装很多 Skill，而不会影响 Claude 的性能——它只在需要的时候才加载对应的内容。

<Info>
  **开放标准**：Anthropic 已经将 Agent Skills 规范作为开放标准发布（[agentskills.io](https://agentskills.io)）。这意味着你为 Claude 创建的 Skill，未来在其他 AI 工具上也可能复用——你的投入不会被锁定在单一平台。
</Info>
