## 适合做成 Skill 的业务场景

以下是一些常见的业务 SOP，它们都非常适合做成 Skill：

| 业务场景 | SOP 包含的关键规则 | 变成 Skill 后的触发方式 |
|---------|------------------|----------------------|
| 写部门周报 | 固定格式、分类逻辑、字数要求 | "帮我做周报" |
| 客户邮件回复 | 品牌语气、必须包含的免责声明、不同场景的回复模板 | "帮我回复这封客户邮件" |
| 会议纪要整理 | 必须记录参会人、决议事项、Action Item 和截止日期 | "帮我整理会议纪要" |
| 发布公告撰写 | 公告格式、审批流程提醒、版本号规范 | "帮我写发布公告" |
| 数据周报生成 | 关注哪些指标、异常值的判断标准、呈现格式 | "帮我分析本周数据" |
| 新人入职检查清单 | 必须完成的步骤、需要申请的权限列表、常见问题 | "帮我生成入职检查清单" |
| 合同/方案审查 | 检查清单、必须包含的条款、常见风险点 | "帮我审查这份方案" |

## 落地路径：谁做什么

Skill 的创建是**业务和技术协作**的过程：

<CardGroup cols={2}>
  <Card title="你（业务人员）需要做的" icon="user">
    1. **梳理 SOP**：把你日常重复做的工作流程写下来——步骤是什么、有什么规则、输出格式是什么样的
    2. **准备模板和参考资料**：比如周报模板、术语表、检查清单
    3. **试用并反馈**：Skill 做好后试用，告诉技术同学哪里不符合预期
  </Card>
  <Card title="技术同学需要做的" icon="code">
    1. 把你的 SOP 转化成 SKILL.md 格式
    2. 配置 Skill 的存放位置和触发规则
    3. 如果需要脚本辅助（比如自动抓取数据），编写 scripts
    4. 版本管理和持续维护
  </Card>
</CardGroup>

## 落地建议

<Steps>
  <Step title="从高频 SOP 开始">
    先挑 1-2 个你**每周都要重复做**的工作，试点做成 Skill。不要一次性把所有 SOP 都做。
  </Step>
  <Step title="保持专注">
    每个 Skill 只做一件事。"周报 Skill"就只做周报，不要混入"会议纪要"的功能。
  </Step>
  <Step title="迭代优化">
    第一版 Skill 不需要完美。用起来之后根据实际效果调整规则和模板。
  </Step>
  <Step title="定期更新">
    SOP 变了，Skill 也要跟着更新。建议每月 review 一次现有的 Skill。
  </Step>
</Steps>

## Skill 需求表模板

如果你的团队准备将 SOP 做成 Skill，可以用下面的模板整理信息交给技术同学：

```markdown
## Skill 需求表

### 基本信息
- Skill 名称：
- 触发词（用户怎么说时触发）：
- 负责人（业务侧）：

### SOP 流程
1. 第一步：
2. 第二步：
3. 第三步：
...

### 输出格式要求
- 格式：[表格 / 文档 / 列表 / ...]
- 示例（附上一份你手动做出的理想输出）

### 关键规则
- 必须做的事：
- 绝对不能做的事：

### 参考资料
- 模板文件：
- 术语表 / 检查清单：
```

<Tip>
  填写这个表的时候，想象你在给一个什么都不知道的新人交代工作——你会怎么说？写下来就对了。
</Tip>
