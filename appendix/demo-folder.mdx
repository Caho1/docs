## 练习文件夹结构

讲师请按以下结构提前准备好练习文件夹：

```
course-demo/
├── README.md                    ← 一份简单的项目说明文档
├── announcement-draft.md        ← 一份待修改的公告草稿
├── meeting-notes.md             ← 一份非结构化的会议笔记
├── data.csv                     ← 一份包含异常值的模拟数据
├── config.json                  ← 一份简单的配置文件
└── docs/
    ├── faq.md                   ← 一份需要检查一致性的 FAQ
    └── process-description.md   ← 一份流程说明文档
```

### 文件内容建议

| 文件 | 内容建议 |
|------|---------|
| `README.md` | 模拟一个内部系统的项目说明，包含一些技术术语，让学员练习"用非技术语言总结" |
| `announcement-draft.md` | 一份产品更新公告草稿，语气比较随意，让学员练习"多风格改写" |
| `meeting-notes.md` | 模拟一份零散的会议笔记（有参会人、讨论内容，但格式混乱），让学员练习"结构化整理" |
| `data.csv` | 模拟销售数据，故意包含一些空值和异常值（如负数金额），让学员练习"数据分析" |
| `config.json` | 简单的配置文件，让学员练习"读配置" |
| `faq.md` | 故意包含一些前后矛盾的回答，让学员练习"一致性检查" |
| `process-description.md` | 一份业务流程描述，让学员练习"通用总结" |

<Warning>
  确保练习文件的内容**对非技术人员友好**——不包含代码，只有文档、数据和配置。所有内容都应该是学员在实际工作中可能接触到的。
</Warning>

## Skill 试点计划模板

如果你的团队准备将 SOP 做成 Skill，可以用下面的模板整理信息交给技术同学：

```markdown
## Skill 需求表

### 基本信息
- Skill 名称：
- 触发词（用户怎么说时触发）：
- 负责人（业务侧）：
- 负责人（技术侧）：

### SOP 流程
1. 第一步：
2. 第二步：
3. 第三步：
...

### 输出格式要求
- 格式：[表格 / 文档 / 列表 / ...]
- 示例（附上一份你手动做出的理想输出）

### 关键规则
- 必须做的事：
  1.
  2.
- 绝对不能做的事：
  1.
  2.

### 参考资料
- 模板文件：
- 术语表 / 检查清单：

### 验收标准
- 怎样算"做得对"：
- 常见的"做错了"的情况：
```

### 填写示例

```markdown
## Skill 需求表

### 基本信息
- Skill 名称：weekly-report
- 触发词：帮我做周报、整理本周工作、生成周报
- 负责人（业务侧）：张三
- 负责人（技术侧）：李四

### SOP 流程
1. 读取用户提供的本周工作记录
2. 按"已完成 / 进行中 / 遇到的问题 / 下周计划"四个分类整理
3. 每个事项用一句话概括
4. 在末尾列出"需要领导关注的事项"

### 输出格式要求
- 格式：Markdown 文档
- 示例：见 templates/weekly-report-example.md

### 关键规则
- 必须做的事：
  1. 风险项必须标注影响范围
  2. 进行中的事项必须标注进度百分比
- 绝对不能做的事：
  1. 不要编造内容
  2. 信息不够时标注"待补充"，不要自己猜

### 参考资料
- 模板文件：templates/weekly-report.md
- 术语表：references/department-terms.md
```
