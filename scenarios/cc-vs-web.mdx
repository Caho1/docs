## 为什么不直接用网页版 AI？

你可能会想："我直接打开 claude.ai，上传文件问问题，不也挺好吗？"

确实，前面几个基础场景（读文档、改文案、查一致性）在网页版 AI 上也能做。但接下来的场景，**只有 Claude Code 能做到，或者做得远比网页版好**。

## 核心差异一览

| 能力 | 网页版 AI (claude.ai) | Claude Code (VS Code 插件) |
|------|----------------------|---------------------------|
| 读取文件 | 手动上传，每次几个 | 直接读取整个文件夹，几十个文件一口气处理 |
| 输出结果 | 显示在聊天框，需手动复制粘贴 | **直接创建/修改文件**，保存到你电脑上 |
| 批量操作 | 不支持，只能一个个来 | 一句话批量处理所有文件 |
| 执行能力 | 有限的代码沙箱 | **在你电脑上运行脚本**，生成图表、Excel 等真实文件 |
| 记忆规范 | 每次新对话都要重新交代背景 | 用 CLAUDE.md 永久记住公司规范和偏好 |
| 跨文件理解 | 上下文受限 | 可以同时理解项目中几十个文件的关系 |

## 一句话总结

<Note>
  **网页版 AI** 像一个"聊天窗口里的助手"——你把东西递给它看，它给你文字回复，你再手动操作。

  **Claude Code** 像一个"坐在你电脑前的助手"——它能直接打开文件夹、读写文件、运行程序、生成报告。你只需要告诉它做什么。
</Note>
