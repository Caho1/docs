## 背景

你有一份重要文档（合同、方案、操作手册），需要检查有没有前后矛盾、遗漏、或表述不清的地方。

人工逐字检查既费时又容易遗漏，让 Claude 帮你做"第一遍检查"，你再做最终确认。

## 推荐话术

```
请仔细阅读 @方案文档.md，帮我检查以下问题：
1. 文档内部有没有前后矛盾的地方？
2. 有没有关键信息缺失或表述模糊的地方？
3. 术语使用是否前后一致？
4. 列出你认为需要我确认的 5 个问题，按优先级排序
```

## 不同类型文档的检查话术

<Tabs>
  <Tab title="产品方案">
    ```
    请检查 @方案文档.md：
    1. 目标和具体措施是否对应？有没有"说了目标但没给方案"的情况？
    2. 时间节点是否合理？有没有前后冲突？
    3. 责任人是否都明确了？
    4. 预算和资源部分是否完整？
    ```
  </Tab>
  <Tab title="操作手册">
    ```
    请检查 @操作手册.md：
    1. 步骤是否完整？有没有跳步或逻辑断层？
    2. 同一个操作在不同地方的描述是否一致？
    3. 有没有步骤的前置条件没有说明？
    4. 新手看了之后能不能照着做？哪些地方可能卡住？
    ```
  </Tab>
  <Tab title="合同/协议">
    ```
    请检查 @合同草稿.md：
    1. 甲乙方的权利义务是否对等？
    2. 有没有条款之间互相矛盾？
    3. 关键日期、金额等数字是否前后一致？
    4. 有没有常见的合同漏洞或缺失条款？
    ```
  </Tab>
</Tabs>

## 使用技巧

<Info>
  Claude 发现的问题不一定都是"真问题"——它可能会过度解读或误判。你的角色是做最终裁判：看看 Claude 列出的问题，判断哪些确实需要修改，哪些可以忽略。
</Info>

**进阶用法——跨文件检查**：

如果你需要检查多份文档之间的一致性，可以用 `@` 引用多个文件：

```
请对比 @需求文档.md 和 @设计方案.md：
1. 需求文档里的每个需求，在设计方案里是否都有对应的设计？
2. 有没有设计方案里有但需求文档里没提到的功能？
3. 用表格列出差异项
```
