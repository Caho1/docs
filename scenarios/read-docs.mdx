## 背景

你收到一份技术文档或 README，里面有很多专业术语，看不太懂，需要快速了解它在讲什么。

这在业务工作中非常常见——技术同学发给你一份项目说明，你需要理解里面的关键信息，但不需要深入技术细节。

## 推荐话术

直接复制到 Claude 对话框即可：

```
请帮我读一下 @README.md，然后回答：
1. 这个项目是做什么的？（用非技术人员能理解的方式说明）
2. 项目的核心模块有哪些？各自负责什么？
3. 有没有我需要特别注意的风险点或限制？
```

## 操作步骤

<Steps>
  <Step title="打开文件夹">
    确保用 `File → Open Folder` 打开了项目文件夹，左侧文件树可见
  </Step>
  <Step title="输入话术">
    在 Claude 对话框输入上面的推荐话术（把 `@README.md` 替换成你实际要读的文件名）
  </Step>
  <Step title="阅读回答">
    阅读 Claude 的回答，注意它是否用了你能理解的语言
  </Step>
  <Step title="追问不清楚的地方">
    如有不清楚的地方，继续追问
  </Step>
</Steps>

## 追问示例

当 Claude 的回答里有你不理解的部分，可以这样追问：

```
第 2 条里提到的 XX 是什么意思？能举个例子吗？
```

```
帮我把这份总结整理成一封邮件，发给我的团队
```

```
这份文档的关键信息，帮我浓缩成 3 句话，我要发给老板
```

<Tip>
  **要点**：不要一次问太多，先让 Claude 总结，确认理解没偏后再追问细节。这就是下一章会讲到的"黄金节奏"。
</Tip>
