## 背景

你有一个口头想法或需求，想让 Claude 帮你写成正式文档。比如产品发布说明、FAQ、需求文档等。

## 推荐话术模板

```
我需要写一份 [文档类型]。背景如下：
[在这里用你自己的话描述需求，不用在意格式，想到什么写什么]

请帮我输出成 [格式要求] 的文档，需要包含：
- [要点 1]
- [要点 2]
- [要点 3]
```

## 具体示例

<Tabs>
  <Tab title="产品发布说明">
    ```
    我需要写一份产品上线发布说明。背景如下：
    我们的数据分析平台这周三要上线新版本，主要改了报表导出功能，
    之前只支持 CSV，现在支持 Excel 和 PDF 了。
    另外修复了一个筛选条件有时候不生效的 bug。

    请帮我输出成正式的发布说明，需要包含：
    - 版本号（帮我用日期格式编一个）
    - 新功能说明（面向用户，不要写技术细节）
    - 问题修复
    - 影响范围（谁需要关注这次更新）
    ```
  </Tab>
  <Tab title="FAQ 文档">
    ```
    我需要写一份客户 FAQ。背景如下：
    我们最近上线了新的会员积分系统，客户经常问：
    积分怎么算的、积分能不能转让、积分会不会过期、
    以前的积分怎么办。

    请帮我输出成 FAQ 格式，需要包含：
    - 每个问题用"Q:"开头，回答用"A:"开头
    - 语气友好专业，适合放在官网
    - 如果有我没想到但客户可能会问的问题，也请补充
    ```
  </Tab>
  <Tab title="会议邮件通知">
    ```
    我需要写一封会议通知邮件。背景如下：
    下周二下午 2 点要开产品评审会，在 3 楼大会议室，
    需要产品、设计、开发三个组的负责人参加，
    主要讨论 Q2 的功能规划。

    请帮我输出成正式的邮件格式，需要包含：
    - 会议主题、时间、地点
    - 参会人员
    - 议程安排（帮我列一个合理的议程）
    - 需要提前准备的材料
    ```
  </Tab>
</Tabs>

## 操作技巧

<Note>
  **核心原则**：用你自己的话描述即可，不需要在意格式。Claude 最擅长的就是把"随意的描述"变成"结构化的文档"。

  你描述得越详细、越接地气，Claude 输出的内容就越贴合你的需求。
</Note>

如果 Claude 生成的文档不太满意，可以这样调整：

```
第二段语气太正式了，改成更口语化的表达
```

```
"影响范围"那部分再具体一些，列出受影响的具体部门
```

```
帮我把这份文档保存到当前文件夹里，文件名叫 release-note-v2.md
```
