title: "四种核心用法"
description: "掌握 Claude Code 插件的四种核心交互方式"
---

Claude Code 插件有四种核心用法，掌握这四种就能覆盖日常工作的绝大多数场景。

## 用法一：直接提问

在 Claude 对话面板中输入问题，就像在 claude.ai 里聊天一样。不同的是，Claude 能**自动感知你当前打开的文件**。

**适合场景**：

- "这份文件在讲什么？帮我总结成 3-5 条要点"
- "这个 JSON 配置文件里每一项是什么意思？"
- "帮我列出这份需求文档里可能有歧义的地方"

<Info>
  Claude 会自动读取你当前打开的文件作为上下文，所以你不需要复制粘贴文件内容。
</Info>

## 用法二：选中文本再提问（最推荐）

这是准确率最高的用法，也是官方重点推荐的使用方式：

<Steps>
  <Step title="选中文本">
    在编辑区**选中一段文字**（鼠标拖选即可）
  </Step>
  <Step title="Claude 自动感知">
    Claude 会自动感知你选中的内容
  </Step>
  <Step title="精确引用（可选）">
    按 `Option+K`（Mac）/ `Alt+K`（Windows）可以插入一个精确的引用标记，告诉 Claude 你在说哪一段
  </Step>
  <Step title="输入需求">
    在对话框输入你的需求
  </Step>
</Steps>

**适合场景**：

- 选中一段文字 → "把这段改成更正式的语气"
- 选中一个段落 → "解释一下这段在说什么"
- 选中一段内容 → "给我两个不同风格的版本：一个正式、一个口语化"

<Tip>
  **为什么最推荐？** 因为你明确告诉了 Claude "我说的是这一段"，它不用猜，回答的针对性和准确度都会更高。
</Tip>

## 用法三：用 @ 引用文件或文件夹

在对话框里输入 `@` 符号，Claude 会弹出文件选择器，让你引用特定的文件或目录。这样 Claude 会完整读取该文件的内容。

**示例**：

```text
@README.md 帮我把这份文档翻译成中文
```

```text
@data.csv 帮我分析这份数据的异常值
```

```text
@docs/ 帮我梳理 docs 目录下所有文档的结构关系
```

<Check>
  `@` 引用比直接提问更精确——直接提问时 Claude 只能看到你当前打开的文件，而 `@` 引用可以让 Claude 读取你指定的任何文件或整个目录。
</Check>

## 用法四：查看并接受/拒绝修改建议

当 Claude 要修改一个文件时，它**不会直接改**，而是展示一个**左右对比视图**（Inline Diff）：

- **左边**：原始内容
- **右边**：Claude 建议的修改
- 你可以逐条****接受（Accept）或拒绝（Reject）****

<Warning>
  这个机制非常重要——它意味着 Claude 的每一个修改都在你的掌控之中。你不需要担心它"悄悄改坏了什么"。
</Warning>

### 操作方式

| 操作                   | 说明                   |
| -------------------- | -------------------- |
| **Accept**（接受）       | 同意 Claude 的修改，文件会被更新 |
| **Reject**（拒绝）       | 不同意修改，文件保持原样         |
| **Accept All**（全部接受） | 一次性接受所有修改            |
| **Reject All**（全部拒绝） | 一次性拒绝所有修改            |

<Tip>
  **新手建议**：先手动逐条确认每个修改，熟悉之后再考虑使用"全部接受"。如果你非常信任 Claude 的修改质量，可以在设置中开启 **Auto-accept**（自动接受），Claude 的修改会实时生效。
</Tip>

## 四种用法总结

| 用法      | 操作方式            | 最适合场景         | 推荐度   |
| ------- | --------------- | ------------- | ----- |
| 直接提问    | 在对话框打字          | 通用问答、快速了解文件   | ★★★   |
| 选中文本再提问 | 先选中文字，再输入需求     | 精确修改、解释特定段落   | ★★★★★ |
| @ 引用文件  | 输入 @文件名         | 处理整个文件、跨文件对比  | ★★★★  |
| 查看修改建议  | 在 Diff 视图中接受/拒绝 | 审核 Claude 的修改 | ★★★★★ |
