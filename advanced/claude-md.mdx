## 什么是 CLAUDE.md

CLAUDE.md 是一个放在项目文件夹里的规则文件。Claude Code 每次启动时会**自动读取**它，就像给 Claude 一份"项目须知"。

你可以把它理解为**给 Claude 的入职手册**——告诉它这个项目的背景、规则和注意事项。

## 技术同学会在里面写什么

| 内容 | 示例 |
|------|------|
| 项目是做什么的 | "这是一个客户管理系统，主要用户是销售团队" |
| 哪些文件不能动 | "config.json 是核心配置，不要修改" |
| 改完要做什么检查 | "修改文档后需要检查中英文术语一致性" |
| 风格要求 | "文档使用 2 空格缩进，标题用中文标点" |

## 对你的影响

如果技术同学已经配好了 CLAUDE.md：

- Claude 对项目的理解会**更准确**
- 回答和建议会**更稳定**（因为有了明确的规则约束）
- 你不需要每次都跟 Claude 重复说明项目背景

**你不需要自己写或维护这个文件**。它是技术同学的职责。

## 写作建议（供技术同学参考）

根据 [Claude Code 官方文档](https://code.claude.com/docs/zh-CN/overview)的建议：

- 控制在 **200 行以内**
- 用 markdown 格式组织，标题分类清晰
- 指令要具体到可验证：比如"使用 2 空格缩进"而不是"格式化代码"
- 避免互相矛盾的规则
- 可以用 `/init` 命令让 Claude 自动生成一份初始版本

<Tip>
  如果你好奇你当前项目有没有 CLAUDE.md，可以在文件树里看看根目录下有没有这个文件，或者直接问 Claude："这个项目有 CLAUDE.md 吗？里面写了什么？"
</Tip>
