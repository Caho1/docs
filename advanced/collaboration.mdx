## 多会话管理

Claude Code 支持同时开多个对话，处理不同任务：

| 操作 | 快捷键（Mac） | 快捷键（Windows） |
|------|-------------|------------------|
| 新建对话 | `Cmd + N` | `Ctrl + N` |
| 切换对话 | 点击对话列表 | 点击对话列表 |

### 什么时候需要多个对话

- 一个对话在改文档，另一个在分析数据
- 当前对话"跑偏了"，开新对话重新开始
- 不同任务的上下文不要混在一起，保持清晰

<Tip>
  切换对话不会丢失之前的上下文。每个对话是独立的，互不影响。
</Tip>

## 与技术同学的协作分工

一个推荐的协作模式：

| 角色 | 负责什么 |
|------|---------|
| **你（业务人员）** | 用 VS Code + Claude Code 处理文档、提需求、整理信息 |
| **技术同学** | 配置 CLAUDE.md、创建 Skill、维护项目工具链 |
| **共同** | 定期沟通哪些 SOP 可以做成 Skill、反馈 Claude 的不足 |

### 一个典型的协作流程

<Steps>
  <Step title="业务人员发现重复工作">
    你发现每周都在重复同样的操作（比如整理周报），而且每次都要跟 Claude 说一遍同样的规则
  </Step>
  <Step title="整理 SOP 给技术同学">
    你把工作流程、规则、输出格式写下来，交给技术同学（可以用[Skill 需求表模板](/skills/examples#skill-需求表模板)）
  </Step>
  <Step title="技术同学创建 Skill">
    技术同学把你的 SOP 转化成 Skill，配置好触发规则
  </Step>
  <Step title="你试用并反馈">
    试用 Skill，把不符合预期的地方反馈给技术同学调整
  </Step>
  <Step title="正式投入使用">
    Skill 调整到位后，你和团队成员都可以直接使用
  </Step>
</Steps>

### 沟通建议

- **定期同步**：建议每两周和技术同学同步一次——哪些 Skill 需要新建、哪些需要调整
- **及时反馈**：发现 Claude 有不符合预期的行为时，及时告诉技术同学，他们可以通过调整 CLAUDE.md 或 Skill 来优化
- **共享发现**：如果你发现了特别好用的提问话术或使用技巧，分享给团队其他人
